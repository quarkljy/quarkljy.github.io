Spring MVC 和Springboot

Posted on 2019-03-12 In SpringBoot Views:
spring 是一个“引擎”，springmvc 是基于spring的一个mvc框架，springboot是基于spring4的条件注册的一套快速开发整合工具集.
1. 前言

spring boot 和spring mvc 其实并没有对比性。spring相当于框架基础，spring mvc是基于spring的一个联通前端和后端的mvc框架，spring boot 是基于spring4的快速开发工具包，具有有自动装配的特性（“约定大于配置”也是Maven的特性）。

2. SpringMVC

Spring MVC提供了一种轻度耦合的方式来开发web应用。它是Spring的一个模块，是一个web框架。通过Dispatcher Servlet, ModelAndView 和 View Resolver，开发web应用变得很容易。

2.1 执行流程

我们假设请求的url为:http://localhost:9999/SpringMVC/input-product，拆分成三部分：

通过分析，如上url表示为：请求位于服务器localhost:9999上的SpringMVC站点的input-product控制器(controller层的某个方法)

  1. DispatcherServlet表示前置控制器，是整个SpringMVC的控制中心。用户发出请求，DispatcherServlet接收请求并拦截请求

  2. HandlerMapping为处理器映射。DispatcherServlet调用HandlerMapping,HandlerMapping根据请求url查找Handler(controller层的某个方法

  3. HandlerExecution表示具体的Handler,其主要作用是根据url查找控制器，如上url被查找控制器为：input-product

  4. HandlerExecution将解析后的信息传递给DispatcherServlet,如解析控制器映射等

  5. HandlerAdapter表示处理器适配器，其按照特定的规则去执行Handler

  6. Handler让具体的Controller执行

  7. Controller将具体的执行信息返回给HandlerAdapter,如ModelAndView

    两种modelandview

  8. HandlerAdapter将视图逻辑名或模型传递给DispatcherServlet

  9. DispatcherServlet调用视图解析器(ViewResolver)来解析HandlerAdapter传递的逻辑视图名

  10. 视图解析器将解析的逻辑视图名传给DispatcherServlet

  11. DispatcherServlet根据视图解析器解析的视图结果，调用具体的视图

  12. 最终视图呈现给用户。

简单概括就是：

  • 根据url请求找到对应的controller方法：DispatcherServlet把请求转发到HandlerMapping，HandlerMapping找到需要的HandlerAdapter，HandlerAdapter再找到需要的Handler,也就是Controller层的某个方法

  • 执行Controller的方法，返回数据: Handler执行（service和dao层的方法调用），得到数据，把数据放到ModelAndView里。

    存数据：

    数据包装到Model，看作是往Map添加Key-Value

    取数据：

    找到Model的key，然后再取出key对应的value

  • 生成具体的页面,展示给用户: Handler返回的数据Model和物理视图View给DispatcherServlet，然后DispatcherServlet把ModelAndView传给ViewReslover（视图解析器），ViewReslover解析后返回具体View（视图），DispatcherServlet根据View进行渲染视图（即将模型数据填充至视图中），DispatcherServlet响应用户。

所用的组件中，前端控制器DispatcherServlet，处理器映射器HandlerMapping，处理器适配器HandlerAdapter，视图解析器ViewResolver都不需要程序员开发，SPring框架提供，但是Handler（具体的Controller）和View（页面）是需要程序员自己来制作的。

以上流程是前后端未分离项目。前后端分离的项目中，Controller层返回对象的Json数据就好了，前端页面自己解析这个对象的Json数据。

3. SpringBoot

Spring Boot 是在强大的 Spring 帝国生态基础上面发展而来，发明 Spring Boot 不是为了取代 Spring ,是为了让人们更容易的使用 Spring，将目前各家公司开发的比较成熟、经得起实际考验的服务框架组合起来，通过 Spring Boot 风格进行再封装屏蔽掉了复杂的配置和实现原理，最终给开发者留出了一套简单易懂、易部署和易维护的分布式系统开发工具包。

3.1 自动装配原理

springboot 以@SpringBootApplication 注解的类的主方法启动，然后按照指定，或者默认的路径进行扫描。取到路径下类上的注解 然后按照注解规定好的规则 进行处理，使用使用默认的配置，application.yml中的配置会覆盖掉默认配置。

  • 对自定义类装配的例子

    方式一

    1. application.yml文件书写相关配置信息
    2. 启动类添加注解@ConfigurationPropertiesScan(basePackages = “com.quark.bean.demobean.config”)，确定扫描路径
    3. 自定义类（实现setter方法）添加注解@ConfigurationProperties(prefix = “demo”)并确定配置信息前缀，类的属性名要跟application.yml文件的属性名一致
    4. 其他类引用，只需要在该对象上添加@Autowired

    方式二

    1. application.yml文件书写相关配置信息
    2. 自定义类上添加注解@Configuration
    3. 属性使用@Value(“${spring.redis.jedis.pool.maxTotal}”)注入
