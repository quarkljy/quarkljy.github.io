1. 前言

有时候需要把本地的项目文件快速部署到云服务上，使用docker部署是个不错方法。因此搭建私仓库私有仓库实现快速部署。

    改成

    这里有个坑，官网说的是改为

    官方文档中写的tcp://127.0.0.1:2375只能本地访问。 需要改成tcp://0.0.0.0:2375。 127.0.0.1是本地接口，只能本地调用。 而0.0.0.0代表本机所有网络接口。

    0.0.0.0:2375表示任何都可以打包和上传镜像，0.0.0.0表示任何ipv4地址，这样设置我们才可以在本地window的idea里用maven插件打包镜像并上传。

    注意，此种方式暴露2375端，容易被黑客扫描到入侵安装挖币程序，因此谨慎使用。生产环境是不会这么部署docker私有仓库，我们自己用是可以的

4. 总结
  1. 先搭建私有仓库，开发端口
  2. maven打包上传镜像。注意，持久化的目录要是空的，未有数据的。
  3. 编写脚本一键部署。注意给脚本的换行要改为 Unix(LF)的\n。脚本要授权。
5. 补充

  1. 在宿主机查看容器内程序在控制台打印的日志命令

  2. 容器的持久化目录不能随便设置，不能是原来就有文件的目录。比如/dev这个目录下有很多其他文件，

    比如maven打包镜像时直接设置持久化目录为/dev是不对的，容器运行会出错。持久化的目录应该是这样的：在容器运行前不存在或者本身就是一个空目录，容器运行后存放项目运行日志数据。

这里的/dev/spring-logs是我们程序运行时候存放日志文件的目录的一部分，容器运行前不存在，容器运行后程序自行创建，也是springboot项目程序存放日志的目录。
