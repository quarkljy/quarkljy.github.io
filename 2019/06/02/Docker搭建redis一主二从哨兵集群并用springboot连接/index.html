Docker搭建redis一主二从哨兵集群并用springboot连接

Posted on 2019-06-02 In redis Views:
部署redis一主二从集群，并开启哨兵监控模式，使用springboot项目本地连接远程集群
1. 前言

redis主从复制可以实现读写分离，哨兵模式实现高可用性。哨兵可以实现监控通知，为客户端提高配置功能。

  • 监控：哨兵不断的检查master和slave是否正常的运行。
  • 通知：当监控的某台Redis实例发生问题时，可以通过API通知系统管理员和其他的应用程序。
  • 自动故障转移：如果一个master不正常运行了，哨兵可以启动一个故障转移进程，将一个slave升级成为master，其他的slave被重新配置使用新的master，并且应用程序使用Redis服务端通知的新地址。
  • 配置提供者：哨兵作为Redis客户端发现的权威来源：客户端连接到哨兵请求当前可靠的master的地址。如果发生故障，哨兵将报告新地址。

3. 创建哨兵的配置文件和docker-compose文件
3.1 哨兵配置文件

使用三个哨兵所以需要配置三份。由于是使用nost模式，所以这三分配置文件的端口不能相同。106.52.114.205服务器公网的ip

sentinel1.conf的端口为26380

sentinel2.conf的端口为26381

sentinel3.conf的端口为26382

细心可以发现，除了端口值不同，其他都相同，这是因为哨兵是通过监控redis主机获取其他从机、其他哨兵的存在的，所以只需要设置自身运行的端口就好了，其他的配置会在哨兵模式运行起来后自动修改。
