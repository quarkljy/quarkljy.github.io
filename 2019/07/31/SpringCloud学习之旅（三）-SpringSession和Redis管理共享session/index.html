SpringCloud学习之旅（三）-- SpringSession和Redis管理共享session

Posted on 2019-07-31 In SpringCloud Views:
SpringSession和Redis管理共享session。深刻理解session。
1. 前言
1.1 session是什么？

session 是客户端与服务器通讯会话跟踪技术。session由服务器端创建。session的使用流程如下：

  1. 浏览器第一次访问服务端，是没有sessionid的，访问服务端后，服务端为了标识这个会话访问，会生成一个新的sessionid（A）和session（A），session（A）保存服务器的内存里。从数据库查询到用户数据id ，name，profile，order等 set到session（A）中，视图解析器从session（A）里get出数据，放到html等网页文件里，再把html文件发送给用户浏览器，用浏览器解析网页文件，得到界面效果，界面有他的id name order profile等数据。同时把sessionid（A）作为cookie发送给浏览器，浏览器在用户电脑的内存中保存这份sessionid（A）字符串–用来做身份标识。总之session（A）相当于JVM里的对象，可以get ,set许多用户的数据，sessionid相当于对象的引用，比如

    sessionid是左边的zhangsan，session是new User()这个对象。

  2. 浏览器第二次访问服务端时，就会带上sessionid（A），服务端根据sessionid（A）到内存里查找对应的session（A）（此时刷新session（A）的过期时间），查找得到，get出用户的id name等数据进一步处理，或者用id name作为查询条件到数据库查询，得到的数据继续set到session（A）里，然后从session（A）里get出数据放入html的文件里，再把html文件发送给用户浏览器，用浏览器解析网页文件，得到界面效果，界面有他的id name order profile等数据。

  3. 用户关闭浏览器时，浏览器会销毁掉那个sessionid（A）的临时cookie（因为它时在内存中的，浏览器关闭了对应的内存也就被电脑清除了）。此时服务端的session（A）对象还是存在的。

  4. 用户打开浏览器再一次访问服务端时，也是没有sessionid的，虽然服务端的session（A）还存在，但是浏览器提供不了sessionid（A）(第3步已经销毁了啊)，所以找不出session（A）。服务端为了标识这个会话访问，会生成一个新的sessionid（B）和session（B）。接下来的就重新循环第1、2、3步。

  5. 在一定时间内没有刷新session（A）的过期时间，那么这个session（A）对象和sessionid（A）就会变成无引用的对象（=null），被服务器端JVM的GC回收，释放响应的内存。

1.2 分布式 session 一致性问题

假设第一次访问服务A生成一个sessionid并且存入cookie中，第二次却访问服务B会在cookie中读取sessionid加入到请求头中，如果在服务B通过sessionid没有找到对应的数据那么它创建一个新的sessionid并返回给客户端,这样并不能共享我们的session,无法达到我们想要的目的–服务端微服务的用户数据可以共享。

2. 分布式session解决方案

使用spring-session和redis来统一管理session，不同的微服务只需要根据sessionid到redis数据库里查询即可得到共享的数据，或者把查询到的数据存放到redis里。

如果把整个集群系统比作一个Java类，那么存放session的redis数据库就是这个类的成员变量，而各个微服务就是这个类的方法。

可能有同学问，这个UUID 一样和不一样有啥含义？

这个UUID代表一次会话中某个微服务M存放到session的数据，访问不同的端口（代表访问不同微服务），得到一样的UUID（代表不同微服务能从session里取到微服务M的存放的数据），说明这个session是被微服务集群共享的且一致的。微服务A可以把自己的从数据库查询到的用户名字name set进这个共享session里，另外一个微服务B可以从session里取出微服务A的数据name,同时它也可以把自己从数据库查询到用户订单数据order set进session里，之后微服务A、微服务C都可以从session里get出这个用户的name和order。

5. 总结

  1. 不要把sessionid和session搞混了。sessionid只是一串不重复的特殊编号，和服务器的session是匹配的。真正存放数据的是服务器内存或者redis里的session对象；

  2. session可以存用户有关的数据，比如名字，订单数据等。
